Welcome

SpaFEDTe is a Template based C++ library for creating Discontinous Finite Element Spaces.

The ultimate (quite ambitious) goal is to provide a framework for the numerical approximation of PDEs. Many high-quality finite element libraries exist and many of them offer a more comprehensive set of features. So, why SpaFEDTe?

SpaFEDTe offers a variety of Discontinuous Finite Element Spaces and allows to easily implement new ones combining (possibly user defined) basis functions, mapping functions and quadrature rules. In addition to standard Reference Frame FE Spaces the library implements Physical Frame FE Spaces, here considered as a way to increase the flexibility and the accuracy of discontinuous Galerkin methods.

Discretizations

  • Discontinuous Galerkin

Finite Element Shapes

  • Segment, 2 and 3 nodes.
  • Triangle, 3 and 6 nodes.
  • Quadrangle, 4, 8 and 9 nodes.
  • Tetrahedron, 4, and 10 nodes.
  • Hexahedron, 8, 20 and 27 nodes.
  • Prism, 6 and 15 nodes.
  • Pyramid, 5 and 13 nodes.

Finite Element Spaces (all FE Shapes supported)

  • Physical frame modal basis functions
    • Arbitrary order orthonormal basis by means of the Modified Gramâ€“Schmidt process.
    • Rescaled rotated monomials.
  • Reference frame modal basis function
    • Generalized tensor product basis functions based on Jacobi polynomials.
    • Monomials.
  • Mapping functions
    • First and second order Lagrangian basis functions.
  • Quadrature rules
    • Arbitrary order Gauss-Jacobi quadrature rules.

FE Data Storage

  • FE related quantities for the whole grid can be stored once and for all in contiguous vectors.
  • FE related quantities can be computed on the fly over each mesh element and mesh face.

Local Matrix Assembly

  • Can be written to be dimension independent (1, 2 and 3D).
  • Can be written to exploit matrix and vectors arithmetic as provided by Eigen3.

Solution visualization

  • Solution output in the parallel vtk format (.pvtu).
  • Cell subdivision depending on the polynomial degree for visualization purposes.

Examples

  • L2 projection
  • Poisson Equation

External Libraries

SpaFEDTe is based on high-quality open-source external libraries

  • MOAB
    • (A Mesh-Oriented datABase), here used for distributed parallel mesh management and mesh partitioning.
  • PETSc
    • (Portable, Extensible Toolkit for Scientific Computation), here used for parallel matrices and vectors data structures and the solution of linear and linearized systems in parallel.
  • Eigen3
    • (a C++ template library for linear algebra: matrices, vectors, numerical solvers, and related algorithms) here used for local vectors and matrices data types, linear algebra, matrix and vector arithmetic and efficient local assembly.
